# 부동산 물건 분석 시스템 구축 기안서

## 1. 개요

- **프로젝트명**: 부동산 물건 데이터 분석 및 자동 보고서 생성 시스템
- **목적**: 부동산 물건 관련 로우 데이터를 입력하면, 지역·유형별 분석 및 유사 물건 분석 결과를 자동으로 생성하여 실제 업무(의사결정, 보고)에 활용할 수 있도록 하는 웹 기반 분석 시스템 구축
- **사용 언어/기술 스택(초안)**:
  - 백엔드: Python, FastAPI
  - 프론트/UI: FastAPI 템플릿(Jinja2) + Bootstrap (웹 UI)
  - 데이터 분석: pandas, numpy, plotly 등
  - 보고서: HTML 보고서 + PDF 변환

## 2. 요구사항 정리 (초안)

### 2.1 입력 데이터(로우 데이터)

- **파일 형식**: 엑셀(.xlsx)
- **주요 컬럼(예시)**:
  - 담당팀
  - 담당자
  - **차주번호**
  - 물건번호
  - 물건유형
  - 물건지 주소
  - 면적
  - 회수 목표가
  - 경매회차
  - 최저입찰가
  - **경매사건번호** (수동 조회용·추후 법원/공공 API 연동 검토)
  - (추가 예정 컬럼은 협의 후 반영)

- **다중 시트(Sheet) 지원**
  - 업로드 엑셀에 **Sheet별로 정해진 시트명**을 사용하면, 각 Sheet의 **동일한 컬럼명**을 기준으로 데이터를 연결(merge/join)하여 **통합 데이터 분석**이 가능하도록 한다.
  - **시트 간 연결 키(공통 키)**는 **차주번호, 물건번호, 담당자**로 한다. 시트별로 이 컬럼이 있으면 해당 키 기준으로 통합 후 지역별·유형별·시계열 분석 등 수행 가능.
  - 예: 시트 "물건마스터", "낙찰이력" 등 사전 정의된 시트명과 위 연결 키를 기준으로 통합.

- **법원 경매 정보 연동**
  - 법원 경매정보는 현재 공식 Open API가 없어, 경매사건번호만으로 자동 수집은 어렵다. 경매사건번호는 수동 조회용 식별자로 두고, 추후 API 또는 원시데이터 개방 시 연동 검토한다.
  - 온비드 등 공공데이터 API는 물건관리번호·공매조건번호 기준으로 활용 가능하므로, 해당 데이터를 다룰 경우 별도 컬럼 및 연동 방안을 검토한다.

### 2.2 분석 및 산출물

- **분석 보고서에 포함될 주요 내용**
  - 지역별 / 물건유형별 통계 및 그래프
    - 건수, 평균/중앙값, 최소/최대, 단위면적당 가격 등 (세부 지표는 협의 후 확정)
  - 유사 물건 분석
    - 동일/유사 지역 및 물건유형 기준으로 유사 물건의 낙찰가 또는 매매가 분석
    - 가격 분포, 최근 트렌드 등 (세부 내용은 협의 후 확정)

### 2.3 출력 형태

- **웹 브라우저에서 분석 결과 확인**
  - 대시보드 형태의 HTML 보고서(표 + 그래프)
- **PDF 출력**
  - HTML 보고서를 기반으로 PDF 파일 생성 및 다운로드 기능 제공

### 2.4 사용 환경

- **접속 방식**: 웹 UI (내부망 또는 지정된 환경에서 접속)
- **사용자**: 부동산 물건을 관리·분석하는 담당자(여러 명이 사용 가능하도록 확장 고려)

## 3. 시스템 구성 (초안)

- **백엔드**
  - FastAPI 기반 REST/웹 애플리케이션
  - 엑셀 파일 업로드, 데이터 전처리, 분석 실행, 보고서 데이터 제공
- **프론트/UI**
  - FastAPI + Jinja2 템플릿을 활용한 서버 사이드 렌더링
  - Bootstrap을 이용한 기본 레이아웃 및 스타일
- **데이터 분석 모듈**
  - pandas를 활용한 엑셀 로딩 및 전처리
  - 지역/유형별 집계, 유사 물건 필터링 및 통계 분석
  - plotly 등을 이용한 그래프 생성 및 웹 페이지 내 시각화
- **보고서 모듈**
  - HTML 기반 분석 보고서 템플릿
  - HTML → PDF 변환(도구는 구현 단계에서 최종 선정: pdfkit+wkhtmltopdf, weasyprint 등 후보)
